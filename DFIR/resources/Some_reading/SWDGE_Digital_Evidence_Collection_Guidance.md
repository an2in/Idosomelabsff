# Digital Evidence Collection

The document recommened analysts to follow these steps (in collecting items that may contain digital evidence):

1. Preparation: Clear communication including details of the investigation, the nature and scope of the potentiel evidence... should be made between the collector and investigate team.

2. Considerations: Prior to collecting digital evidence, consider collecting and preserving traditional forensic
evidence (e.g., fingerprint, DNA, trace). When collecting digital evidence, considerations should be taken to preserve the integrity of the data being acquired.

3. Data Integrity: As the name suggests.

4. Preparation: This prep is for the digital storage media. Data sanitization of storage media should be performed to
prevent cross-contamination.

5. Security: To maintain data integrity, security is instrumental.

6. Search: Clear out non-essential people. Photograph, sketch, and label the area before touching anything. Look for everything that stores data or grants access (post-its with passwords, encryption keys, network diagrams). If it’s off, leave it off. If it’s on (or in standby): check for destructive processes. Capture RAM/volatile data if possible, isolate it from the network, then strictly pull the power cable. Stop and call in specialists if hitting live disk encryption, virtual machine hosts, SANs, or IoT devices. (What if this "call" get into a loop:DDD?)

7. Acquisition and Live Acquisition: Acquire the data in a technique that is the least invasive possible. Examiners should be aware that any live collection of data can alter and create evidence and should be documented.

8. Documentation
